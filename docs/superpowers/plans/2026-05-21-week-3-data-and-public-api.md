> Executed inline 2026-05-20/21. Use `superpowers:subagent-driven-development` only if rerunning. Most tasks already complete.

**Goal:** Backend now owns the canonical data model (coaches, packages, success stories, plus lead-capture and Stripe-related tables for upcoming weeks). Public read endpoints serve coaches / packages / success stories. Frontend's `/our-team` and `/success-stories` consume the API at request time (with revalidation) and fall back to the hand-written catalog when the API isn't reachable.

**Architecture:** SQLAlchemy 2.0 async + Alembic for the schema. Pydantic v2 schemas with `from_attributes=True`. Thin FastAPI routes that delegate to a service per resource. Frontend uses Next.js Server Components for SSR/ISR; `src/lib/api.ts` is a single boundary that tries the API and gracefully falls back to static lib data on any failure. Generated TypeScript types from `/openapi.json` live in `src/lib/api-client/` and are gitignored.
